## Environment quirk: Vulkan shader build

This worktree uses a newer Vulkan SDK where `active` is now a
reserved word in GLSL. Two shaders in `native/vulkan/shaders/`
(`gdn_scan_step_f32.comp`, `gdn_scan_multi_token_f32.comp`)
fail to compile against it. The CLAUDE Code agent renamed the local
variable to `act_lane` to run tests, then reverted before
committing — the rename is **not** part of any Step 45 commit.

If subsequent agents hit the same glslc error, apply
`sed -i 's/\bactive\b/act_lane/g'` to those two files locally. The
proper fix belongs in Phase 10 (Qwen3MoeHybrid Vulkan path).
