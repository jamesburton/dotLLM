## Codebook embedding strategy

The IQ1_S grid is **2048 entries x 8 packed signed-int8 ternary values**
({-1, 0, +1}). On CPU it lives as `ulong[2048]` (16 KB). On GPU each shader
embeds it as `const uint[4096]` (each ggml uint64 split LE into low/high
uint pair, 16 KB per shader). Compiled SPV blobs are 24-30 KB — well under
any practical UBO budget; tested and works fine on RDNA 3.5 Strix Halo.

Llama.cpp's vulkan backend uses a packed 2-bit-per-coordinate variant
(`iq1s_grid_const[1024]`, 4 KB) loaded into shared memory at workgroup
init. We did not adopt that scheme because:
  - 16 KB const-array embedding compiles cleanly and runs fast on Strix Halo.
  - Decoding from the verbatim ggml uint64 grid means the GPU oracle
    matches the CPU oracle bit-for-bit (0 ULP dequant parity), with no risk
    of grid-derivation drift across backends.
  - Test latency is acceptable (full IQ1_S suite runs in ~1 minute including
    per-block-fixture brute-force quantisation).

## Pre-existing test issues observed (not regressed by this work)

- `VulkanGdnDecayF32KernelTests.Launch_MatchesCpuReference(seqLen=17, nVHead=32)`
  fails with **maxUlp=5** (vs 4-ULP threshold), 1/544 elements. Pure
  tolerance flake. The decay shader does not use `active`/`laneActive` and
  is not affected by the rename in commit 1.

- `VulkanGdnScanMultiTokenF32KernelTests.Launch_MatchesCpuReference(*)`
  fails with **maxUlp=12572**, ~3766/16384 elements (5/5 cases). This is a
  functional failure, not a tolerance issue. Reproduces with the exact SPV
  blob committed in upstream `665f228` — i.e. the test was failing on
  Strix Halo before this work began. Confirmed by checking out the original
  IQ4 baseline SPV byte-for-byte, same failure mode. Not related to the
  IQ4 cherry-picks or to IQ1_S; tracked as a pre-existing GDN multi-token
  scan issue on this hardware.

Both issues are Strix Halo-specific. They were *not* visible at base
`feature/qwen3.6` @ `0524276` because the multi-token scan test couldn't
even build (missing SPV blob; commit 1 is the prerequisite to make it
runnable on a clean clone with current Vulkan SDK).

quantisation" (IQ1_S).
