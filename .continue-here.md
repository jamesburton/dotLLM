### Two structural bugs fixed

- **`745b091`** — HF safetensors Llama-family RoPE: `Norm` → `NeoX`. HF uses
  `rotate_half` (halves); GGUF converter permutes Q/K weights so GGUF stays
  on `Norm`. TinyLlama 3/5 → 5/5 argmax; 30× tighter drift. Also fixes
  Mistral/Mixtral/DeepSeek-V2 (latent, no real-weight tests for those yet).
- **`576ea91`** — DeepSeek shared-expert loader recognises fused-plural
  naming (`mlp.shared_experts.{gate,up,down}_proj`, no numeric index) per
  HF's `DeepseekV2MoE.__init__`. Shared-expert branch was silently no-op'ing
  on all 26 MoE layers. 1/5 → 5/5 argmax; 91× tighter drift.

- `MlaExpandedKvState` — expanded per-head K/V cache (oracle, `Tight` intra-dotLLM).
- `MlaLatentKvState` — latent `[kv_lora_rank + qk_rope]` cache + `W_UK`

## Open items (pick any; all independent)

### 1. CPU perf gap: MMQ-style fused dequant+matmul (HIGHEST ROI)

Expected 2–3× decode speedup. Current flow in `MatMul.cs` / `GemmInterleaved`:
dequantize Q8_0 into F32 scratch → `TensorPrimitives.Dot` per row. llama.cpp
MMQ fuses dequant-in-register during matmul. **Profile first**
(`dotnet-trace collect --format speedscope` against `InferenceBenchmarks`)
to confirm the hot path before coding.

### 2. CPU perf gap: tiled GEMM with cache blocking (MEDIUM ROI)

1.5–2× prefill. Current `MatMul.cs` is row-major matvec; needs 8×8 or
16×16 tile blocking.

### 3. Vulkan LLM kernels (biggest long-term win on this box)

llama.cpp Vulkan on the iGPU does **392 tok/s decode**; we do 38. Port
order documented in `CONTINUE_ON_CUDA.md` §Reciprocal. ~3-4 weeks for a
complete Llama-family Vulkan forward. CUDA kernels in
`native/kernels/*.cu` transpile mechanically to GLSL/SPIR-V.

### 4. Long-context YaRN validation

`e7b4aef` wired the math + unit tests. `DeepSeekV2Lite_LongContext_LogitsMatchPyTorchReference`
stub skips until `deepseek-v2-lite-longctx-reference.json` is generated.
Command in the test's XML doc. Requires ~32 GiB bf16 HF inference on
~4500 tokens (~30 min wall).

### 5. Secondary HF divergences (latent for current tests)

- **`routed_scaling_factor`** — parsed in `MoeConfig`, ignored in
  `MoeSwiGluMlp`. V2-Lite=1.0 no-op; V2-full/V3 use ≠1.0 when
  `norm_topk_prob=false`. Add `topk_weight *= routed_scaling_factor`.
- **V3 `topk_method=group_limited_greedy`** — `MoEGate.forward` lines
  439-460 of `C:/temp/dotllm-deepseek-v2-lite/modeling_deepseek.py`.
  Not implemented; V2 uses `greedy` which we handle.

### 6. ROCm/HIP on Linux (blocked on gfx1151 driver)

Try WSL2 + `rocm/dev-ubuntu-24.04:6.4-complete`. If `rocminfo` lists
`gfx1151`, rebuild `native/hip/kernels/rmsnorm.hip` with explicit
`--offload-arch=gfx1151` in `build.sh`.

### 7. llama.cpp cross-library harness

`C:/Development/OllamaBenchmarks/scripts/benchmark_throughput_openai.py`
