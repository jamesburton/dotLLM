## Decisions made / locked in

- **MoE dispatch model = Option B** (sparse indexed-expert matmul kernel with per-row index lookup + fused weighted scatter). Rejected the host-readback per-(t,slot) loop (Option A — slow on prefill) and the dense-all-experts-with-mask (Option C — 8× compute waste on real Mixtral). Production MoE engines do similar.
- **MoE shared-expert storage = per-expert separate buffers** (NOT packed into a bank). Shared experts are few (1-2 per layer); the matmul kernel reads from offset 0; packing would need a sub-buffer view or a new weight-offset push constant. Trade-off documented in `MoeLayerBuffers` docstring.
- **MoE tiled indexed matmul heuristic**: route to tiled when `n ≥ 32 && m % TILE_M == 0`. Decode (n ≤ 16) stays on scalar; prefill uses tile.
- **Sigmoid-gated shared expert: fused kernel** (`moe_sigmoid_gated_add_f32`) instead of separate `sigmoid_inplace` + scaled-add. One dispatch, 3 buffers, push consts (seqLen, hiddenSize). Reads gate-logit per token, applies sigmoid, fuses into the per-row weighted in-place add.
- **NemotronH F32-only quant first cut**. All projection weights dequantised to F32 at upload. Q8_0 + K-quants deferred — kernels exist, just need an upload path. Documented in `VulkanNemotronHWeights` class docstring.
- **NemotronH test fixture: internal `BuildFromPrebuiltWeights` factories** on both CPU `NemotronHTransformerModel` and `VulkanNemotronHTransformerModel`. Sidesteps the GGUF-loader-only constraint (NemotronH has no LoadFromSafetensors). Tests own the unmanaged weight memory directly.
- **Mamba-3 qk_norm: Option A wrapper** (no new shader). The math is `RmsNorm` with `rowCount = T*nGroup, n = dState`. Cleaner than cloning the rmsnorm shader. Confirmed `RmsNormF32Kernel.Record` is alias-safe for in-place input/output.
- **Both Mamba-2 (NemotronH) AND Mamba-3 in scope**. User correction: there are more Mamba-2 models in production (state-spaces, NVIDIA Nemotron-H releases) than Mamba-3 models, and the kernels for both share the foundation (silu_inplace, conv1d_causal). Both architectures land via this branch.
- **Mamba-3 MIMO scan: yPerRank + Streaming variants out of scope** for the kernel commit (`36e0c0a`). They are CPU-side fixture-comparator + chunked-decode features; not needed for production MIMO forward.
