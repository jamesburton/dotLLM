Stage D2 (next): safetensors weight loading into unmanaged memory
(NativeMemory.AlignedAlloc, MemoryMappedFile). Open questions that Stage
D1 surfaced and Stage D2 must resolve before any forward pass:

1. A_log is absent from ib-ssm/mamba3-370M-10BT safetensors. The
   reference registers self.A_log as nn.Parameter([nheads]) and uses
   A = -exp(A_log) per forward. HF checkpoint has no A_log. Decide:
     (a) derive A from A_floor (0.0001) as fixed per-head constant, or
     (b) A_log was serialized under a different name (e.g. folded into
         in_proj bias — but use_bias=false), or
     (c) the checkpoint saved with strict=False and dropped it.
   Run a tiny PyTorch load-and-print probe against the actual weights to
   confirm which. Do NOT proceed to a forward pass until A is resolved.

2. F32 storage despite config.dtype=bfloat16. Safetensors header probe
   shows every tensor stored as F32; config says bfloat16. Stage D2 must
   mmap as F32 and treat config.dtype as misleading.

3. B_bias / C_bias shape on HF is [nheads, 1, d_state] (3-D with
   singleton middle axis) even in SISO mode. Reference SISO is
   [nheads, d_state] (2-D). Squeeze on load.

4. No MLP. HF checkpoint has no per-layer SwiGLU (only {mixer, norm}).
   Mamba3Block as currently composed does not include an MLP either, so
   this is consistent — but the reference does interleave MLPs. Anyone
   porting from the reference impl for sanity checks must drop the MLP.

5. RoPE fraction = 0.5, not 1.0. The reference assumes all state pairs
   rotate; a fraction < 1.0 implies partial-RoPE. Mamba3DataRoPE may
   need a new parameter or a separate call for the non-rotated half.
