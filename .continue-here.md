<decisions_made>

- Used VikramKarLex/mamba3-minimal as reference (Albert Gu-endorsed, ~800 line pure PyTorch)
- Strict test tolerances AbsTol=1e-6 / RelTol=1e-5 (user asked for "a bit stricter" than 1e-4/1e-3; observed F32 drift is ~6e-7 so this leaves ~2× margin)
- Inlined the paper's "two-SSD decomposition" into a single-pass scalar scan kernel (equivalent result, half the memory traffic vs calling SSD twice)
- Kept Mamba3SelectiveScan scalar-only (correctness first; SIMD deferred)
- Block composer allocates scratch per-call (new float[]) — a long-lived scratch
  pool analogous to NemotronHForwardState is a simple follow-up
- Three kernel alias-safety bug fixes from the predecessor branch (SiLu, SwiGLU,
  ReluSquared) are shared and stay in PR #135, not re-introduced here
</decisions_made>
