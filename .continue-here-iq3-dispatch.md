## Real-GGUF parity gate

The task brief also mentions a `Skip.IfNot(File.Exists)` real-GGUF
parity gate. With dispatch wired in (any one host suffices), this is a
single test that:
1. Skip unless `~/.dotllm/test-cache/IQ3-model.gguf` exists.
2. Load CPU and Vulkan paths.
3. Run a fixed prompt through both.
4. Assert top-K logit agreement with the IQ2-like tolerance the IQ2 family uses
   (≤ 1e-2 abs, ≤ 5e-3 rel on final logits).

This is gated, so the test stays skipped in CI but runs locally when
the model is cached. Pattern mirrors the existing IQ4_XS / IQ2_S real-
GGUF gates.

## Risk surface

- **Constructor arg order**: each host constructor already has 50-100
  parameters. Slotting 5 new params (4 kernels + codebooks) anywhere
  except next to the existing IQ2 params will scatter call sites.
  Place IQ3 params **immediately after** the IQ2 family params at
  every call site (constructor + factory `Create`). Verify by
  recompiling — the C# compiler catches misordered arguments.
- **Dispose ordering**: dispose kernels before the shared codebook.
  IQ3 kernels hold a reference to `Iq3Codebooks` only when
  `ownsCodebooks=false`; with the shared host pattern the host owns
  the codebook and the kernels never dispose it. Mirror exactly the
  IQ2 dispose order.
- **InvalidateDescriptorCache after weight rebind**: the existing IQ2
  pattern invalidates the descriptor cache on every weight reupload
  (e.g. LoRA swap). IQ3 must do the same — the codebook handle
  doesn't change but the weight buffer handle in the descriptor set
  does.

## Sanity tests after each host

After each host's wiring lands:
```
dotnet test tests/DotLLM.Tests.Unit --filter "FullyQualifiedName~Iq3" --no-build
dotnet test tests/DotLLM.Tests.Unit --filter "FullyQualifiedName~Vulkan{Host}TransformerModel" --no-build
```
The first ensures kernel parity still holds; the second ensures the
host's full forward pass still works (no regression on F16/F32/etc
paths). Don't ship a host without both.
